WORKING CONVENTIONS
- Ownership-first propagation (adopted 22 Jul 2026, from the content-audit
  seam pattern — the sharpest of the three structural findings). Before
  editing any topic, look it up in R30 and identify the OWNING document.
  Edit the owner FIRST, then every document that delegates to it, in the
  SAME session — never the reverse. R42-then-R10 is the failure mode that
  produced F38 (R42 corrected while R10 §3, which R42 delegates to, still
  carried superseded fire doctrine); R10-then-R42 would not have produced
  it. SESSION-CLOSE COMPLETENESS ASSERTION: for each topic touched, list
  the documents that delegate to it and assert each was visited, or
  explicitly deferred to a numbered open item — same control class as the
  COUNTIF baseline on the working papers. An R30 OWNER-flag column, which
  would convert this discipline from an act of vigilance into a lookup, is
  approved in principle and DEFERRED until after the Karoo loop.
- Numeric traceability, self-audit, and verify-before-you-report (adopted
  22 Jul 2026 from the content-audit programme close). (a) Any specific
  numeric specification introduced into a library document must be
  traceable to Facts, the owning R-doc, or a manual — a plausible number
  reads as authority and is the most dangerous thing an assistant can add
  to a field reference (F65, a self-introduced defect). (b) A self-audit
  of prior-session Claude edits is a standing step in every audit session
  — it is the only control that caught F65. (c) A candidate finding is
  VERIFIED before it is written down; a false finding costs the owner's
  trust in the whole register (F74's dropped candidate). (d) Completeness
  checklists are written BEFORE the file is opened, never after.
- Experimental / pre-EX-9 route planning (adopted 13 Jul 2026): the expedition
  route is developed as an EXPERIMENTAL DRAFT that lives OUTSIDE the library
  until EX-9 ratifies it — kept in ordinary chats, NOT the project, so a fluid
  draft is never mistaken for committed fact. Source of truth = the working
  brief (LC76_Experimental_Route_OptionA_WORKING.docx / .md); the owner edits
  in place, and a "CHANGES THIS ROUND" list at the top of the doc is the
  reconciliation signal on re-upload (read it first, reconcile the body against
