Keeping This Index Current

R30 uses a hand-curated embedded JSON index. When new documents are added or existing ones gain major new sections, the index below needs updating.

How to update the topic index

  1. Open LC76_Cross_Reference_Index.html in a text editor.
  2. Find the const INDEX_DATA block near the bottom of the <script> section.
  3. Each entry in files[] has an id, title, desc, and topics[] array.
  4. To add a new document: add a new object to files[] with its R-number, filename, description, and topics.
  5. To add topics to an existing document: append to its topics[] array. Each topic is { t: "topic text", tags: ["tag1","tag2"] }.
  6. Also add new documents to the categories[] block so they appear in the By Topic view.
  7. Update the BUILT date constant at the top of the script.
  8. Commit to GitHub — the file is self-contained, no build step needed.
Tip: When building a new R-document, ask Claude to append its topic list to INDEX_DATA at the same time. Keep the topics array factual and specific — "diff breather" not "differential".
