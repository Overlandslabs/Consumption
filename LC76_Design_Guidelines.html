S1 Two Design Systems Reference Docs · Calculators

The library uses two distinct design systems. They share a philosophy and a few tokens but are not interchangeable. Always identify which system a file belongs to before editing.

FontsBebas Neue + IBM Plex Mono + OswaldOswald only
Primary accent--amber2 #e07800--clay #C4572A / --ochre #D4963A

S4 Layout Width · Spacing · Structure
Reference doc content must be full browser width max-width:100% with percentage padding only: padding:28px 5% 80px. Never use a fixed pixel max-width on the content area. Headers, footers, sticky nav, section headers, and xref bars are always 100% width.
Use html{"{"}overflow-x:hidden{"}"} — not body body{"{"}overflow-x:hidden{"}"} breaks position:sticky in Safari. Always apply overflow-x:hidden to the html element.
ElementRule
Sticky nav containerposition:sticky; top:0; z-index:100; padding-top:env(safe-area-inset-top)
Masthead + TOC barMust be wrapped together in a single .sticky-nav-unit div — never separate sticky elements

TOC bar backgroundbackground:var(--steeldk) + border-bottom:2px solid var(--amber)

S6 Navigation TOC · Scroll · iOS PWA
⚠ TOC links must be <a> tags — not <span> iOS does not reliably fire touch events on <span> elements in standalone (home screen) PWA mode. All TOC links must use <a href="javascript:void(0)"> with inline onclick. This is the only pattern confirmed to work across browser and iOS standalone. 
<!-- CORRECT — works in browser and iOS standalone PWA -->
<a href="javascript:void(0)" onclick="document.getElementById('s1').scrollIntoView({behavior:'smooth'});return false;" class="toc-link">1. Section</a>

<!-- WRONG — <span onclick> unreliable on iOS standalone -->
<span class="toc-link" onclick="scrollToSection('s1')">1. Section</span>

<!-- WRONG — function named scrollTo() collides with window.scrollTo -->
function scrollTo(id){ ... }
Dynamic Scroll MarginPrevents content hiding behind sticky nav
/* Set scrollMarginTop dynamically — never hardcode it.
   Sticky nav height varies by screen size and safe-area insets. */
function updateScrollMargins(){
  const navH = document.querySelector('.sticky-nav-unit').offsetHeight;
  document.querySelectorAll('.sec-header').forEach(el => {
    el.style.scrollMarginTop = (navH + 4) + 'px';
  });
}
updateScrollMargins();
window.addEventListener('resize', updateScrollMargins);
IntersectionObserver — Active TOC State
const sections = ['s1','s2','s3','s4','s5','s6'];
const tocLinks = document.querySelectorAll('.toc-link');

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if(entry.isIntersecting){
      const idx = sections.indexOf(entry.target.id);
      tocLinks.forEach((l, i) => l.classList.toggle('active', i === idx));
    }
  });
}, {rootMargin: '-30% 0px -60% 0px'});

sections.forEach(id => {
  const el = document.getElementById(id);
  if(el) observer.observe(el);
});

S7 Print CSS Colour forcing · A4 · Section 16
⚠ All backgrounds must be explicitly forced in print CSS Browsers strip background colours in print mode by default. Without print-color-adjust:exact AND explicit !important colour declarations on every dark element, all dark bars, badges, and alert boxes will print white.

S8 PWA & iOS Home screen · Service worker · Manifests
RequirementValue / Rule
viewport metawidth=device-width,initial-scale=1.0,viewport-fit=cover
apple-mobile-web-app-capableyes — enables full-screen standalone mode
apple-mobile-web-app-status-bar-styleblack-translucent — matches dark masthead through status bar

ManifestReference docs share manifest-ref.json. Each calculator has its own: manifest-power.json, manifest-water.json, manifest-tyre.json, etc.
Service worker strategyNetwork-first — always fetch latest; cache as offline fallback only. Never cache-first for HTML files.
Horizontal overflowhtml{overflow-x:hidden} — use html, NOT body. Body overflow-x:hidden breaks Safari sticky.

S9 Do / Don't Common mistakes
✓ Do

Use <a href="javascript:void(0)" onclick="...scrollIntoView..."> for all TOC links

Use html{overflow-x:hidden} — never body

Wrap masthead and TOC bar in a single .sticky-nav-unit

Set scrollMarginTop dynamically in JS — never hardcode it

Begin every print block with *{-webkit-print-color-adjust:exact!important;print-color-adjust:exact!important}

Force every dark background colour with !important in print CSS

Use --steeldk for .task-spec text — not amber/orange

Use IBM Plex Mono for all label, badge, spec, and metadata text

Use Bebas Neue for all section and subsection titles

Keep reference doc content full-width — max-width:100%

Use bar as primary pressure unit; PSI is secondary

Test in iOS Safari browser AND after adding to home screen

Delete and re-add home screen shortcut after any JS change

✗ Don't

Name any function scrollTo() — collides with window.scrollTo

Use <span onclick> for TOC links — unreliable on iOS standalone

Use body{overflow-x:hidden} — breaks Safari sticky

Separate masthead and TOC into independent sticky elements

Hardcode scroll-margin-top — it won't adapt to screen size changes

Omit print-color-adjust:exact — everything prints white

Use amber/orange for .task-spec text — too low contrast

Mix reference doc tokens into calculator files or vice versa

Use a fixed pixel max-width cap on reference doc content

Use href="#id" anchor links for TOC — ignore sticky offset

Use prompt() or alert() — blocked in standalone PWA

Use addEventListener + data-target pattern in reference docs — use inline onclick on <a> tags instead
